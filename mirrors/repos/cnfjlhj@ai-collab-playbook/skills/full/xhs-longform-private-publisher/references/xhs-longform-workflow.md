# 小红书长文私密发布参考流程

这个参考文件记录的是“已有 Markdown 原文 + 已有插图”的真实发布流程，重点是：不要改写原文、图片按位插入、用一键排版、最后以私密方式发布。

## 1. 入口与页面类型

- 长文入口：`https://creator.xiaohongshu.com/publish/publish?target=article`
- 笔记管理页：`https://creator.xiaohongshu.com/new/note-manager`

如果页面已经落在编辑页或管理页，不必强制回入口；优先判断当前页是否已经处在可安全继续的状态。

## 2. 建议的准备命令

先用脚本构建载荷：

```bash
python scripts/build_payload.py <markdown_path> --image-dir <image_dir> --output /tmp/xhs_payload.json
```

然后优先用自动化发布脚本执行：

```bash
python scripts/publish_longform.py --payload /tmp/xhs_payload.json --validate-only
python scripts/publish_longform.py --payload /tmp/xhs_payload.json --publish
```

其中：

- `--validate-only` 只校验载荷并打印执行计划，不启动浏览器
- 不带 `--publish` 时会真正打开浏览器并填充到发布前页，但不会点最终发布
- 带 `--publish` 时会在发布后跳到笔记管理页，查找标题前缀和 `仅自己可见`，并输出截图与摘要文件

理想的载荷结构如下：

```json
{
  "title": "原文标题",
  "parts": ["<p>...</p>", "<p>...</p>"],
  "images": ["<img1>", "<img2>"]
}
```

其中 `parts` 的长度应当是 `images` 长度加一。

## 3. 页面识别

长文编辑页通常会同时出现以下特征：

- 标题输入区域，常见提示语为“填写标题会有更多赞哦”
- 长文富文本编辑器，常见 DOM 为 `div.tiptap.ProseMirror`
- 右侧或下方的预览区，能看到页码和标题预览
- 页面底部有“发布”按钮

不要把短描述框误认为长文正文。短描述框即使可编辑，也不等于长文正文已经写入成功。

## 4. 插入标题与正文

标题：

- 把载荷中的 `title` 填入标题输入框

正文：

- 聚焦 `div.tiptap.ProseMirror` 或等价的富文本编辑器
- 逐段插入 `parts`
- 建议使用浏览器端的 `document.execCommand('insertHTML', false, html)` 方式插入 HTML

已验证过的思路：

```javascript
const editor = document.querySelector('div.tiptap.ProseMirror');
editor.focus();
document.execCommand('insertHTML', false, htmlPart);
```

如果编辑器升级导致这条路径失效，优先寻找平台自身的富文本命令，或者退回到“逐段复制到可识别的富文本区域”的做法。

## 5. 图片插入

正文和图片的顺序应当是：

`part1 -> image1 -> part2 -> image2 -> ... -> partN`

操作原则：

- 每插完一段正文，再插对应图片
- 上传完成后，确认光标重新回到正文可继续输入的位置
- 再继续插下一段正文

一个已知经验是：工具栏中的 `button.menu-item` 有一个按钮对应“图片”，某次实测中它位于从 `0` 开始计数的第 `8` 个位置。

这个下标只能当“最后已知经验”，不能当稳定契约。优先顺序应该是：

1. 文本或提示语能直接识别“图片”按钮时，优先用文本识别
2. 如果工具栏有图标提示或 aria 标签，优先用语义定位
3. 只有在页面无语义信息时，才退回到按钮顺序经验

## 6. 一键排版

必须在“全部正文和图片都已插完”后，再点击“一键排版”。

原因：

- 如果过早排版，后续再补内容很容易破坏模板结构
- 如果排版前内容还没稳定，用户看到的最终效果常常会很乱

默认接受平台当前提供的模板，不要额外改写文案内容。

## 7. 私密设置

发布前必须确认“仅自己可见”已经打开。

检查方式：

- 编辑页中能看到“仅自己可见”状态
- 发布后，笔记管理页的最新一条卡片上也能看到 `仅自己可见`

如果两边状态不一致，以发布后的管理页为最终验收准。

## 8. 发布后验收

至少做两类验证：

### 8.1 管理页验证

回到笔记管理页，确认最新一条笔记具备：

- `仅自己可见`
- 标题前缀正确
- `发布于 <当前时间附近>`
- 状态是 `审核中` 或 `已发布`

管理页是最关键的验收页，因为它最直接反映“是否发出去、是否私密、是不是最新一条”。

### 8.2 编辑页或预览页验证

如果需要进一步确认，可点开最新笔记：

- 看标题是否对
- 看预览区是否有页码与内容
- 看“仅自己可见”是否仍在

注意：某些时候编辑页的短描述框会显示为空，但这不代表长文正文丢失；要优先看预览区和管理页。

## 9. 常见问题与对策

### 问题 1：管理页返回后显示空列表或数量为 0

这常常是页面短暂骨架屏或异步刷新问题。等待 `仅自己可见`、标题前缀或最新时间戳出现后再判断。

### 问题 2：图片错位

优先怀疑“上传图片后光标位置没回到预期区域”。如果错位明显，重新新建一篇比在坏稿里修补更稳。

### 问题 3：正文变成原始 Markdown

说明插入的是纯文本而非 HTML。重新走 `build_payload.py`，确认 `parts` 是 HTML，再插入。

### 问题 4：标题在管理页被截断

这是正常的列表展示行为。用标题前缀和发布时间联合确认即可。

### 问题 5：按钮定位失效

优先改用文本定位、语义定位或页面快照重新识别；不要盲信历史下标。

## 10. 重试建议

最多重试 3 次，并记录每次失败的原因。

- 第 1 次失败：查清是正文插入问题、图片顺序问题，还是私密设置问题
- 第 2 次失败：新开一篇，避免延续坏状态
- 第 3 次失败：缩小自动化范围，只保留最稳定的自动步骤，其余关键环节人工核对

## 11. 最小交付证据

至少保留：

- 一张笔记管理页截图
- 截图中能看到 `仅自己可见`
- 截图中能看到标题前缀和发布时间

如果还保留了编辑页或预览页截图，会更利于后续复盘。

## 12. 自动化脚本约定

`publish_longform.py` 当前的关键定位策略如下：

- 长文编辑器：`div.tiptap.ProseMirror`
- 长文标题框：`textarea[placeholder="输入标题"]`
- 排版后标题框：`input[placeholder="填写标题会有更多赞哦"]`
- 一键排版按钮：文本 `一键排版`
- 最终发布按钮：文本 `发布`
- 图片按钮：优先用语义定位，失败时回退到 `button.menu-item` 的历史经验位次

脚本不是把这些定位器当永恒契约，而是按“语义优先、经验兜底”的顺序尝试。如果未来平台改版，应先更新这里，再更新脚本。
