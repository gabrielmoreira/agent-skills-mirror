## 评分系统概述

PaperAgent 使用多维评分系统对论文进行全面评估，包括：

1. **四维评分体系**：工程应用价值、网络架构创新、理论贡献、结果可靠性
2. **Date-Citation 权衡机制**：公平对比不同发表时间论文的影响力
3. **最终综合评分**：加权计算四维评分和影响力评分

**设计目标**：
- 全面评估论文的多个维度
- 公平对比不同时期的论文
- 突出高价值论文

## 四维评分体系

### 1. 工程应用价值 (Engineering Application)

**评分范围**：1-10分

**评估维度**：
- 解决实际工程问题的能力
- 工业级验证程度
- 部署可行性与效率优势
- 实际应用场景的广度

**评分标准**：

| 分数 | 描述 |
|------|------|
| 9-10 | 解决重大工业问题，有完整工业级验证，可直接部署应用 |
| 7-8 | 解决实际工程问题，有较好的实验验证，有明确应用前景 |
| 5-6 | 有一定工程价值，但验证有限或应用场景较窄 |
| 3-4 | 工程价值有限，主要是方法验证，缺乏实际应用 |
| 1-2 | 纯理论研究，无工程应用价值 |

**评估要点**：
- 是否解决了实际工程问题？
- 是否有工业级验证？
- 部署可行性如何？
- 效率优势是否明显？

### 2. 网络架构创新 (Architecture Innovation)

**评分范围**：1-10分

**评估维度**：
- 架构设计的新颖性
- 模块和机制的创新
- 与现有架构的对比优势
- 设计合理性

**评分标准**：

| 分数 | 描述 |
|------|------|
| 9-10 | 提出全新架构范式，开创性设计，可能引领新方向 |
| 7-8 | 显著架构创新，重要模块或机制创新，优于现有架构 |
| 5-6 | 有一定创新，在现有架构基础上有改进 |
| 3-4 | 微小创新，主要是现有架构的组合或微调 |
| 1-2 | 缺乏架构创新，直接应用现有架构 |

**评估要点**：
- 是否提出了新的架构范式？
- 架构设计是否新颖？
- 与现有架构相比有何优势？
- 设计是否合理？

### 3. 理论贡献 (Theoretical Contribution)

**评分范围**：1-10分

**评估维度**：
- 是否提出新的数学框架
- 是否证明重要定理
- 是否建立新的理论连接
- 理论深度与严谨性

**评分标准**：

| 分数 | 描述 |
|------|------|
| 9-10 | 提出新的数学框架、证明重要定理、建立新的理论连接，具有开创性理论贡献 |
| 7-8 | 对现有理论有重要深化，提供新的理论洞察，改进理论边界 |
| 5-6 | 有一定理论分析，但深度有限 |
| 3-4 | 理论分析较浅，主要是引用现有理论 |
| 1-2 | 缺乏理论分析，纯实验验证 |

**评估要点**：
- 是否提出了新的数学框架？
- 是否证明了重要定理？
- 是否建立了新的理论连接？
- 理论深度如何？

**理论贡献识别要点**：
1. 是否提出新的数学形式化框架？
2. 是否证明新的定理或建立新的理论边界？
3. 是否建立不同领域之间的理论连接？
4. 是否提供对现有方法更深入的理论理解？
5. 是否解决长期存在的理论问题？

### 4. 结果可靠性 (Reliability)

**评分范围**：1-10分

**评估维度**：
- 实验设计严谨性
- 开源代码与数据
- 结果可复现性
- 实验结果支持度

**评分标准**：

| 分数 | 描述 |
|------|------|
| 9-10 | 实验严谨，代码和数据完全开源，结果可完全复现 |
| 7-8 | 实验设计合理，代码或数据部分开源，结果基本可信 |
| 5-6 | 实验基本合理，缺乏开源支持，结果有待验证 |
| 3-4 | 实验设计有缺陷，缺乏开源支持，结果存疑 |
| 1-2 | 实验设计有问题，结果不可信 |

**评估要点**：
- 实验设计是否严谨？
- 是否开源代码和数据？
- 结果是否可复现？
- 实验结果是否充分支持假设？

## Date-Citation 权衡机制

### 设计目标

公平对比不同发表时间论文的影响力：
- 奖励最新发表的高价值论文
- 认可早期发表并已被广泛引用的论文
- 避免因发表时间不同导致的不公平

### 论文年龄计算

```python
from datetime import datetime

publication_date = datetime.fromisoformat(publication_date)
current_date = datetime.now()

# 计算年龄（天数）
age_days = (current_date - publication_date).days

# 转换为月数（使用标准月份天数）
age_months = age_days / 30.44

# 计算引用密度
citation_density = citation_count / age_months if age_months > 0 else 0
```

### 年龄边界定义

| 论文年龄 | 月数范围 | 说明 |
|---------|---------|------|
| 最新论文 | ≤ 3个月 | age_months ≤ 3 |
| 中期论文 | 3-24个月 | 3 < age_months ≤ 24 |
| 成熟论文 | > 24个月 | age_months > 24 |

### 调整因子规则

#### 最新论文（≤ 3个月）

- 统一给予 +0.2 奖励
- **说明**：最新论文引用数尚未积累，统一给予奖励

#### 中期论文（3-24个月）

| 引用数 | 调整因子 |
|-------|---------|
| ≥ 50 | +0.5 |
| 20-49 | +0.3 |
| 10-19 | +0.2 |
| < 10 | +0.1 |

#### 成熟论文（> 24个月）

| 引用数 | 调整因子 |
|-------|---------|
| ≥ 200 | +0.5 |
| 100-199 | +0.4 |
| 50-99 | +0.3 |
| 20-49 | +0.2 |
| < 20 | +0.0 |

### 引用密度奖励

**适用范围**：所有年龄段

| 引用密度 | 奖励 |
|---------|------|
| ≥ 10次/月 | 额外 +0.2 |
| 5-10次/月 | 额外 +0.1 |
| < 5次/月 | 无额外奖励 |

### 调整因子上限

- **上限**：+1.0
- **说明**：最终影响力评分不超过10分

## 最终评分计算

### 计算公式

```python
# 四维基础评分
four_dim_score = (
    engineering_value +
    architecture_innovation +
    theoretical_contribution +
    result_reliability
) / 4

# 最终综合评分
final_score = four_dim_score * 0.9 + impact_score * 0.1
```

### 权重说明

| 评分项 | 权重 | 说明 |
|-------|------|------|
| 四维基础评分 | 0.9 | 综合评估论文质量 |
| 影响力评分 | 0.1 | 评估论文影响力 |

**权重设计理由**：
- 四维评分（0.9）：主要评估论文本身的质量
- 影响力评分（0.1）：适当考虑论文的影响力，避免过度依赖引用数

## 评分示例

### 示例 1：最新高价值论文

**论文信息**：
- 发表时间：2024年12月（2个月前）
- 基础影响力：9分
- 引用数：5

**评分计算**：
- 四维基础评分：8.5分
- 论文年龄：2个月（最新论文）
- 调整因子：+0.2（最新论文奖励）
- 引用密度：5/2 = 2.5次/月（无额外奖励）
- 调整后影响力：9.0 + 0.2 = 9.2分
- 最终综合评分：8.5 × 0.9 + 9.2 × 0.1 = 8.57分

### 示例 2：中期高引用论文

**论文信息**：
- 发表时间：2023年6月（18个月前）
- 基础影响力：8分
- 引用数：45

**评分计算**：
- 四维基础评分：8.0分
- 论文年龄：18个月（中期论文）
- 调整因子：+0.3（中期论文引用奖励）
- 引用密度：45/18 = 2.5次/月（无额外奖励）
- 调整后影响力：8.0 + 0.3 = 8.3分
- 最终综合评分：8.0 × 0.9 + 8.3 × 0.1 = 8.03分

### 示例 3：成熟经典论文

**论文信息**：
- 发表时间：2022年1月（36个月前）
- 基础影响力：8分
- 引用数：180

**评分计算**：
- 四维基础评分：8.0分
- 论文年龄：36个月（成熟论文）
- 调整因子：+0.4（成熟论文引用奖励）
- 引用密度：180/36 = 5次/月（额外+0.1）
- 调整后影响力：8.0 + 0.4 + 0.1 = 8.5分
- 最终综合评分：8.0 × 0.9 + 8.5 × 0.1 = 8.05分

### 示例 4：最新论文但影响力一般

**论文信息**：
- 发表时间：2024年11月（3个月前）
- 基础影响力：6分
- 引用数：2

**评分计算**：
- 四维基础评分：6.0分
- 论文年龄：3个月（最新论文）
- 调整因子：+0.2（最新论文奖励）
- 引用密度：2/3 = 0.67次/月（无额外奖励）
- 调整后影响力：6.0 + 0.2 = 6.2分
- 最终综合评分：6.0 × 0.9 + 6.2 × 0.1 = 6.02分

## 评分标准详细说明

### 工程应用价值评分详细标准

#### 9-10分：解决重大工业问题

**特征**：
- 解决重大工业问题
- 有完整工业级验证
- 可直接部署应用
- 效率优势明显

**示例**：
- DeepONet：首次实现算子通用近似定理的实用架构，可学习任意非线性连续算子
- FNO：傅里叶神经算子，在多个工业场景中验证

#### 7-8分：解决实际工程问题

**特征**：
- 解决实际工程问题
- 有较好的实验验证
- 有明确应用前景
- 部署可行性较高

**示例**：
- Transolver：Transformer架构用于PDE求解，在多个数据集上验证

#### 5-6分：有一定工程价值

**特征**：
- 有一定工程价值
- 验证有限
- 应用场景较窄
- 部署可行性一般

#### 3-4分：工程价值有限

**特征**：
- 工程价值有限
- 主要是方法验证
- 缺乏实际应用
- 部署可行性低

#### 1-2分：无工程应用价值

**特征**：
- 纯理论研究
- 无工程应用价值
- 主要是理论探索

### 网络架构创新评分详细标准

#### 9-10分：提出全新架构范式

**特征**：
- 提出全新架构范式
- 开创性设计
- 可能引领新方向
- 理论支持充分

**示例**：
- DeepONet：首次提出分支网络+主干网络架构，基于算子通用近似定理
- FNO：傅里叶神经算子，首次将傅里叶变换引入神经算子

#### 7-8分：显著架构创新

**特征**：
- 显著架构创新
- 重要模块或机制创新
- 优于现有架构
- 设计合理

**示例**：
- CViT：连续视觉Transformer，建立离散实现与连续算子之间的理论桥梁
- Transolver：Transformer架构用于PDE求解，显著优于传统方法

#### 5-6分：有一定创新

**特征**：
- 有一定创新
- 在现有架构基础上有改进
- 改进效果明显
- 设计合理

#### 3-4分：微小创新

**特征**：
- 微小创新
- 主要是现有架构的组合或微调
- 改进效果有限

#### 1-2分：缺乏架构创新

**特征**：
- 缺乏架构创新
- 直接应用现有架构
- 无明显改进

### 理论贡献评分详细标准

#### 9-10分：开创性理论贡献

**特征**：
- 提出新的数学框架
- 证明重要定理
- 建立新的理论连接
- 理论深度与严谨性高

**示例**：
- CViT：提出连续视觉Transformer，建立了离散实现与连续算子之间的理论桥梁
- DeepONet：基于通用近似定理的算子学习理论框架

#### 7-8分：重要理论深化

**特征**：
- 对现有理论有重要深化
- 提供新的理论洞察
- 改进理论边界
- 理论分析深入

**示例**：
- FNO：证明了傅里叶层在函数空间中的通用近似性

#### 5-6分：有一定理论分析

**特征**：
- 有一定理论分析
- 但深度有限
- 主要是引用现有理论

#### 3-4分：理论分析较浅

**特征**：
- 理论分析较浅
- 主要是引用现有理论
- 缺乏新的理论洞察

#### 1-2分：缺乏理论分析

**特征**：
- 缺乏理论分析
- 纯实验验证
- 无理论贡献

### 结果可靠性评分详细标准

#### 9-10分：实验严谨

**特征**：
- 实验严谨

- 实验结果充分支持假设

**示例**：

#### 7-8分：实验设计合理

**特征**：
- 实验设计合理
- 代码或数据部分开源
- 结果基本可信
- 实验结果基本支持假设

#### 5-6分：实验基本合理

**特征**：
- 实验基本合理
- 缺乏开源支持
- 结果有待验证

#### 3-4分：实验设计有缺陷

**特征**：
- 实验设计有缺陷
- 缺乏开源支持
- 结果存疑

#### 1-2分：实验设计有问题

**特征**：
- 实验设计有问题
- 结果不可信
